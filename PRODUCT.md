# Product

## Register

product

## Users

เจ้าหน้าที่กลุ่มงานสถิติข้อมูลและสารสนเทศที่ต้องบันทึกสถิติการขอดูภาพจากกล้องวงจรปิดควบคู่กับใบคำร้องกระดาษเดิม ใช้งานทั้งบนโต๊ะทำงานและหน้างานผ่านมือถือหรือแท็บเล็ต เพื่อเพิ่มคำร้อง ค้นหา แก้ไข แนบหลักฐาน และออกรายงานอย่างรวดเร็ว

## Product Purpose

ระบบนี้ทำให้การบันทึกคำร้อง CCTV เป็นระเบียบ ค้นหาได้ ออกรายงานได้หลายมิติ และลดงานซ้ำจากเอกสารกระดาษ โดยยังไม่แทนที่กระบวนการราชการหลัก เป้าหมายความสำเร็จคือเจ้าหน้าที่เพิ่มคำร้องใหม่ได้ในเวลาไม่เกิน 30 วินาที รายงานถูกต้อง และ UI น่าเชื่อถือพอสำหรับใช้งานจริงบน Vercel/Neon ในช่วงทดลอง

## Brand Personality

รวดเร็ว น่าเชื่อถือ สวยคมชัด น้ำเสียงของระบบต้องสุภาพ กระชับ และเป็นภาษาราชการที่ไม่แข็งเกินไป ประสบการณ์ควรให้ความรู้สึกเหมือนเครื่องมือภายในที่ออกแบบมาอย่างตั้งใจ ไม่ใช่ฟอร์มฐานข้อมูลธรรมดา

## Anti-references

ไม่ควรเหมือน landing page หรือ template SaaS ที่มี hero ใหญ่เกินจำเป็น ไม่ใช้การ์ดซ้อนการ์ด ไม่ใช้สีจัดหรือ gradient เพื่อความตื่นตาอย่างเดียว ไม่ทำ workflow ซับซ้อนแบบ ticket/approval system และไม่เก็บข้อมูลส่วนบุคคลเกินจำเป็น

## Design Principles

1. เริ่มงานได้ทันที: หน้าแรกต้องเป็น dashboard สำหรับทำงานจริง พร้อมทางลัดไปเพิ่มคำร้อง ค้นหา และรายงาน
2. เร็วก่อนลึก: ฟอร์มหลักต้องสั้น ชัด และเหมาะกับนิ้วบนมือถือ ส่วนรายละเอียดรองค่อยเปิดเมื่อจำเป็น
3. ตัวเลขต้องเชื่อถือได้: เลขคำร้อง รายงาน และตัวกรองต้องสื่อสารสถานะและข้อผิดพลาดชัดเจน
4. สวยแบบเครื่องมือจริง: ใช้ความประณีตจาก typography, spacing, state และ data visualization ไม่ใช่ของตกแต่ง
5. เผื่ออนาคตโดยไม่ถ่วงปัจจุบัน: โครงสร้างต้องรองรับการนำเข้าข้อมูลย้อนหลัง สิทธิ์ผู้ใช้ รายงานเพิ่ม และ workflow เพิ่มภายหลัง แต่ MVP ต้องยังเบา

## Accessibility & Inclusion

ตั้งเป้า WCAG AA สำหรับข้อความและปุ่มหลัก รองรับ reduced motion มี focus state ชัดเจน ใช้ touch target ที่เหมาะกับมือถือและแท็บเล็ต หลีกเลี่ยงการสื่อสารด้วยสีเพียงอย่างเดียว และต้องอ่านภาษาไทยได้สบายในสภาพแสงหน้างาน
